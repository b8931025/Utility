當專案丟出下列例外時，是因為Web.config設定不適合所在環境的問題。可以用以下兩種方試解決
1.將  管理->服務->ASP.NET State Service  打開
2.將mode的StateServer改成InProc  
<configuration>
<system.web>
<sessionState mode="InProc "  stateConnectionString="tcpip=127.0.0.1:42424" cookieless="false" timeout="600"  />
</system.web>
</configuration>  

'/BotDev' 應用程式中發生伺服器錯誤。

無法將工作階段要求送至工作階段狀態伺服器。請確定已經啟動 ASP.NET State Service 且用戶端與伺服器端的通訊埠都相同。如果伺服器是在遠端電腦上，請檢查 HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\aspnet_state\Parameters\AllowRemoteConnection 的值，確定它是否接受遠端要求。

描述: 在執行目前 Web 要求的過程中發生未處理的例外情形。請檢閱堆疊追蹤以取得錯誤的詳細資訊，以及在程式碼中產生的位置。
